# Spreewölfe Berlin — Community Match Calendars

> **⚠️ This is an unofficial community project.**
> It is **not** created by, affiliated with, or endorsed by
> [Spreewölfe Berlin e.V.](https://www.spreewoelfe.de/)
> All schedule data is sourced from publicly available information on
> [spreewoelfe.de](https://www.spreewoelfe.de/spielplaene/) and
> [ishd.de](https://www.ishd.de/).
> For official information, always refer to the club's website.

This project provides iCalendar (`.ics`) files for the
[Spreewölfe Berlin](https://www.spreewoelfe.de/) inline skaterhockey
match schedules so that fans, parents, and players can easily subscribe
to game dates in their calendar apps.

## How to Generate the Calendar Files

The iCal files are generated by scraping the schedule pages on
[spreewoelfe.de](https://www.spreewoelfe.de/spielplaene/).
This should be re-run once per year when new schedules are published
(typically early in the calendar year).

### Prerequisites

```bash
pip install requests beautifulsoup4
```

### Generate All Teams

```bash
cd spreewoelfe/
python fetch_schedules.py --year 2026 --output .
```

### Generate a Single Team

```bash
python fetch_schedules.py --team herren-i --year 2026 --output .
```

### Options

```text
--team SLUG      Fetch only one team (e.g. herren-i, damen, jugend-u-16)
--year YEAR      Season year (default: current year)
--output DIR     Output directory (default: current directory)
--skip-discover  Don't auto-discover teams; use the built-in list only
```

## Locations Database

The file [`locations.json`](locations.json) contains a growing database of
all known venue addresses used by the teams. It is updated automatically
whenever the fetch script runs. You can also edit it manually.

## Yearly Checklist

This project is intended to be updated once per season. Here's the process:

1. Wait for the new schedules to be published on
   [spreewoelfe.de/spielplaene](https://www.spreewoelfe.de/spielplaene/)
   (usually early in the year)
2. Run `python fetch_schedules.py --year YYYY --output .`
3. Verify the generated `.ics` files (open them in a calendar app)
4. Update this README: change the year numbers and QR code URLs
5. Commit and push
